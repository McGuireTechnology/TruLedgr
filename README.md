## Repository Structure

Subprojects

| Directory   | Description                                                                                   |
|------------|-----------------------------------------------------------------------------------------------|
| `api/`     | Backend API code and tests for the TruLedgr platform.                                         |
| `dashboard/` | Web dashboard frontend for managing and visualizing data.                                    |
| `apple/`   | Apple platform apps, including iOS, iPadOS, and macOS clients, built with Swift and Xcode.    |
| `android/` | Android app, built with Kotlin and Android Studio.                                            |
| `docs/`    | Project documentation, powered by MkDocs, including developer guides and technical docs.      |
| `www/`     | Static website or landing page assets for the project.                                        |
| `old/`     | Archived or legacy files and previous versions, kept for reference.                           |

Root-level files:

| File         | Description                                                        |
|--------------|--------------------------------------------------------------------|
| `README.md`  | Project overview, setup instructions, and repository structure.     |
| `LICENSE`    | The open source license for the project.                           |
| `SECURITY.md`| Security policy and vulnerability reporting instructions.           |

These subprojects and files are organized to separate backend, frontend, mobile, documentation, and legacy code, making the repository easy to navigate and maintain.
